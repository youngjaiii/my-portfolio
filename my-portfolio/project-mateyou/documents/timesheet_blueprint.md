## 시스템 개요

### 핵심 원칙

- **"출근을 기록하는 앱이 아니라, 사람의 행동을 통제하고 추적하는 관리 시스템"**
- 권한 분리 명확
- 상태 모델 단순
- UI가 역할에 맞게 분리
- 모바일 우선 반응형 디자인
- 컴포넌트 재사용성 최대화

### 기술 스택

- **프론트엔드**: React + TypeScript + TanStack Router
- **상태 관리**: Zustand + React Query
- **UI 라이브러리**: shadcn/ui (Tailwind CSS)
- **백엔드**: Supabase (PostgreSQL + Realtime)
- **스타일**: Tailwind CSS (모바일 우선)

## 권한별 페이지 구조

### 라우트 구조

```
/timesheet
├── / (대시보드 - 역할별 자동 분기)
├── /admin (어드민 상세 관리)
├── /admin/settings (어드민 설정)
├── /manager (매니저 상세 관리)
└── /partner-plus (파트너+ 상세)
```

### 권한별 접근 가능 페이지

| 권한 | 대시보드 | 상세 관리 | 설정 |
|------|---------|----------|------|
| **어드민** | ✅ | `/timesheet/admin` | `/timesheet/admin/settings` |
| **파트너 매니저** | ✅ | `/timesheet/manager` | ❌ |
| **파트너M** | ✅ | `/timesheet/manager` (제한적) | ❌ |
| **파트너+** | ✅ | `/timesheet/partner-plus` | ❌ |

### 1. `/timesheet` - 공통 대시보드

#### 1.1 파트너+ 대시보드

**레이아웃 구조**
```
┌─────────────────────────────────┐
│  Timesheet 대시보드              │
├─────────────────────────────────┤
│                                 │
│      [현재 상태 표시]            │
│      ┌─────────────┐            │
│      │  출근 중    │            │
│      └─────────────┘            │
│                                 │
│  ┌─────────────────────────┐   │
│  │     출근 요청            │   │
│  └─────────────────────────┘   │
│                                 │
│  [근태 이력] (최근 5개)         │
│  ┌─────────────────────────┐   │
│  │ 2024-01-15 09:00 출근   │   │
│  │ 2024-01-15 12:00 휴게   │   │
│  │ 2024-01-15 13:00 재개   │   │
│  │ 2024-01-15 18:00 퇴근   │   │
│  └─────────────────────────┘   │
└─────────────────────────────────┘
```

**상태별 UI**

**OFF (미출근)**
- 상태 배지: 회색 배경, "미출근" 텍스트
- 액션 버튼: "출근 요청" (Primary, Large)
- 이력 섹션: 최근 근태 이력 표시

**WORKING (출근 중)**
- 상태 배지: 초록색 배경, "출근 중" 텍스트
- 액션 버튼:
  - "휴게 요청" (Secondary)
  - "퇴근 요청" (Primary)
- 실시간 정보: 출근 시간, 경과 시간 표시

**BREAK (휴게 중)**
- 상태 배지: 노란색 배경, "휴게 중" 텍스트
- 액션 버튼:
  - "휴게 종료" (Primary)
  - "퇴근 요청" (Secondary)
- 실시간 정보: 휴게 시작 시간, 경과 시간 표시

**요청 대기 중**
- 상태 배지: 파란색 배경, "승인 대기 중..." 텍스트
- 액션 버튼: "승인 대기 중..." (Disabled)
- 요청 정보: 요청 시간, 가게, 매니저 표시

**컴포넌트 구성**
- `StatusDisplay` - 상태 배지
- `AttendanceRequestSheet` - 요청 시트
- `AttendanceHistoryCard` - 이력 카드
- `RealTimeClock` - 실시간 시계

## API 명세

### 인증
모든 API는 Supabase 인증 토큰이 필요합니다.

### 엔드포인트

#### 역할 관련
- `GET /timesheet/role` - 사용자 역할 조회
- `GET /timesheet/status` - 현재 근태 상태 조회
- `GET /timesheet/pending` - 승인 대기 요청 확인

#### 요청 관련
- `POST /timesheet/requests` - 근태 요청 생성
- `PATCH /timesheet/requests/:id/approve` - 요청 승인
- `PATCH /timesheet/requests/:id/reject` - 요청 반려
- `GET /timesheet/requests/pending` - 승인 대기 요청 목록

#### 기록 관련
- `GET /timesheet/records` - 근태 기록 조회
- `GET /timesheet/records/working` - 출근 중 파트너+ 목록
- `PATCH /timesheet/records/:id` - 근태 기록 수정

#### 감사 로그
- `GET /timesheet/audit-logs` - 감사 로그 조회
- `POST /timesheet/audit-logs` - 감사 로그 기록 (내부)

#### 가게 관리
- `GET /timesheet/stores` - 가게 목록 조회
- `POST /timesheet/stores` - 가게 생성
- `PATCH /timesheet/stores/:id` - 가게 수정
- `DELETE /timesheet/stores/:id` - 가게 비활성화

#### 매니저 관리
- `GET /timesheet/managers` - 매니저 목록 조회
- `POST /timesheet/managers` - 매니저 지정
- `DELETE /timesheet/managers/:id` - 매니저 해제

#### 파트너+ 관리
- `GET /timesheet/partner-plus` - 파트너+ 목록 조회
- `POST /timesheet/partner-plus` - 파트너+ 추가
- `DELETE /timesheet/partner-plus/:id` - 파트너+ 삭제

#### 정산
- `GET /timesheet/settlements` - 정산 데이터 조회
- `GET /timesheet/settlements/export` - 정산 데이터 엑셀 다운로드
