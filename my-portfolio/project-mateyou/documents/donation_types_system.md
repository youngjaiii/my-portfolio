# 도네이션 타입 시스템 설계

## 1. 개요

스트림(보이스룸/비디오룸)에서 다양한 형태의 도네이션을 지원합니다.
호스트는 도네이션 컨트롤 센터에서 도네이션을 관리(읽기, 처리, 스킵)할 수 있습니다.

## 2. 도네이션 타입

| 타입 | 설명 | 보이스룸 | 비디오룸 |
|------|------|:-------:|:-------:|
| `basic` | 일반 도네이션 (기본) | ✅ | ✅ |
| `mission` | 미션 도네이션 (특정 요청) | ✅ | ✅ |
| `video` | 영상 도네이션 (유튜브 링크) | ❌ | ✅ |

### 2.1 일반 도네이션 (basic)
- 기존 하트 후원과 동일
- 금액 선택 → 후원 완료
- 채팅에 후원 메시지 표시

### 2.2 미션 도네이션 (mission)
- 후원자가 미션 메시지 작성
- 호스트가 미션 수행 여부 결정
- 미션 수락/거절 가능

### 2.3 영상 도네이션 (video)
- 유튜브 링크 입력
- 호스트가 재생 여부 결정
- 비디오룸에서만 사용 가능

## 3. 도네이션 상태

| 상태 | 설명 |
|------|------|
| `pending` | 대기 중 (호스트 확인 전) |
| `playing` | 재생 중 (영상 도네이션) |
| `completed` | 처리 완료 |
| `skipped` | 스킵됨 |

## 4. DB 스키마 변경

### 4.1 stream_donations 테이블 확장

```sql
-- 기존 컬럼에 추가
ALTER TABLE stream_donations
ADD COLUMN donation_type TEXT DEFAULT 'basic' CHECK (donation_type IN ('basic', 'mission', 'video')),
ADD COLUMN status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'playing', 'completed', 'skipped')),
ADD COLUMN mission_text TEXT,          -- 미션 도네이션: 미션 내용
ADD COLUMN video_url TEXT,             -- 영상 도네이션: 유튜브 URL
ADD COLUMN video_title TEXT,           -- 영상 도네이션: 영상 제목
ADD COLUMN video_thumbnail TEXT,       -- 영상 도네이션: 썸네일 URL
ADD COLUMN processed_at TIMESTAMPTZ,   -- 처리 완료 시간
ADD COLUMN processed_by UUID;          -- 처리한 호스트 ID
```

## 6. 주요 기능

### 6.1 도네이션 큐

- 대기 중인 도네이션을 순서대로 표시
- 타입별 필터링 (전체/미션/영상)
- 실시간 업데이트 (Realtime)

### 6.2 호스트 액션

| 액션 | 설명 |
|------|------|
| 읽기 | TTS로 도네이션 내용 읽기 |
| 완료 | 도네이션 처리 완료 표시 |
| 스킵 | 도네이션 스킵 (포인트는 유지) |
| 재생 | 영상 도네이션 재생 (비디오룸) |

### 6.3 자동 처리

- 일반 도네이션: 이펙트 표시 후 자동 완료
- 미션/영상 도네이션: 호스트 수동 처리 필요
